The Pipeline

Stitch (design tool)
  │  MCP: get_screen, list_screens
  ↓
./stitch-sync/sync.sh        ← deterministic, no LLM
  ├── design-tokens.json      → Tailwind @theme
  ├── html/                   → reference per screen
  ├── structure/              → diffable JSON outlines
  └── screenshots/            → visual reference
  │
  ↓  implement in code
  │
  ↓  MCP: edit_screens        ← push changes back
Stitch (updated)

Bidirectional. Deterministic scripts. Git-versioned.

Talking Points (1 min)

Key decision: the sync pipeline is pure shell/curl/jq. No AI in the loop for sync. LLM is only for generating and editing screens. This makes it repeatable and version-controllable.
