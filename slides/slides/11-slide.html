Bidirectional Sync

We renamed "Arcade" → "Scene" in our codebase. Then pushed it back to Stitch:

# Push rename to Stitch via MCP
mcp__stitch__edit_screens(
  prompt="Rename all instances of Arcade to Scene"
)

# Pull updated designs back
./stitch-sync/sync.sh all

# Verify in code
git diff stitch-sync/structure/
# dashboard.json: "My Arcades" → "My Scenes" ✓

Design stays in sync with code. Not the other way around.

Talking Points (1 min)

This is the pipeline in action. Code decision → push to design → pull back → verify structurally. The design reflects our implementation decisions, not vice versa. That's unusual — most teams do design → code, never code → design.
