Dark Mode from Variant Analysis

Generated a dark Editor variant → extracted the color mapping → CSS variable override:

/* Light tokens (default) */
@theme {
  --color-surface: #f9f9ff;
  --color-on-background: #141b2b;
}

/* Dark override — .dark on <html> */
.dark {
  --color-surface: #0f1219;
  --color-on-background: #e2e4f0;
}

Every element using token colors (bg-surface, text-on-background) gets dark mode for free. No dark: classes needed.

Talking Points (1.5 min)

This is the power of design tokens + CSS custom properties. Stitch gave us the dark palette via generate_variants. We extracted it into a .dark {} override. Every element using token-based colors automatically switched. Only hardcoded colors (slate-*, white) needed manual fixes.
