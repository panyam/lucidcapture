No Component Tree

Stitch outputs flat HTML. No component hierarchy, no semantic structure API.

What we wantedWhat Stitch gives
Component tree with propsFlat HTML with Tailwind classes
Diffable design changesScreenshots (volatile) or HTML (noisy)
Semantic element IDsHTML comments like <!-- TopNavBar -->

The fix: Built extract-structure.sh — parses HTML into diffable JSON outlines capturing headings, buttons, links, layout classes.

Talking Points (1.5 min)

This is Stitch's biggest gap. Screenshots are volatile (pixel-level styling = false positives), HTML diffs are noisy (Tailwind class reordering). We needed a middle layer — structural outlines that capture WHAT changed without the noise of HOW it looks.
