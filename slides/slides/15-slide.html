Edit = Copy (Screen Proliferation)

edit_screens creates new screens instead of editing in place.

  • Renamed "Arcade" → "Scene" on 6 screens → got 6 new copies
  • Generated 3 layout variants → 3 new screens with same title
  • After a few sessions: 29 screens, 14 were duplicates

The fix: Tombstoning

// stitch-sync/tombstones.json — checked into git
{
  "deleted": [
    { "id": "e0c62e39...", "title": "Arcade Details",
      "reason": "Replaced by Scene Details", "deletedAt": "2026-03-23" }
  ]
}

Sync script skips tombstoned IDs. Teams share deletion decisions via git.

Talking Points (1.5 min)

No delete_screen API exists. No rename API. Screen IDs in Stitch UI are hard to find. Built tombstoning as a coordination mechanism — when someone deletes a screen, the tombstone file tells everyone else it was intentional. The sync script auto-prunes stale tombstones.
