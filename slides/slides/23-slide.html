Key Takeaways

  • For solo devs: Screenshots + LLM is faster. Use Stitch when you need system-level consistency
  • For teams: MCP makes design a shared, programmable system — not a screenshot you email
  • Design tokens are the real value — they drive Tailwind themes, enable dark mode for free, keep stacks in sync
  • Build the tooling layer — sync scripts, structure extraction, tombstoning. Stitch alone isn't enough
  • Stitch is early — no components, no themes, edit=copy. But MCP is the right abstraction
  • The future: Design as API. Multiple agents collaborating on a shared design system

Talking Points (2 min)

This is the slide people will remember. Hit each point clearly. The core message: MCP makes design tools programmable infrastructure. That's the paradigm shift, not the AI generation.
