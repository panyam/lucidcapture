Principle #1

"Don't use LLM tokens for a job a shell script can do."

🐌

Prompting each time

45 sec per file
Non-deterministic
Burns tokens on repetition

→

⚡

Script (written once by the LLM)

2 sec total
Deterministic (curl + jq)
Same input → same output, every time

The LLM writes the tool. The tool does the work. No tokens burned on repetition.

Key moment (1 min)

"The LLM is great at writing the sync script. But using it to BE the sync script is not the best use of tokens or time."

"More importantly — how do you ensure determinism? The script calls curl and jq directly. The Stitch REST API, not the LLM, processes the data. Same API call, same response, same file. Every time."

"If the LLM is in the loop, you get slightly different output each run. Different JSON formatting, different field ordering, maybe it summarizes something. The script gives you exact reproducibility — which means git diff actually works."
