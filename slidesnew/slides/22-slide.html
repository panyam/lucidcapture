Tombstoning: .gitignore for Design Screens

Stitch has no delete_screen API. Edits create copies. Variants multiply. After a few sessions:

  • screens multiply, many are duplicates
  • All with the same title
  • No way to tell which is "current"

The fix: a checked-in tombstone file.

// stitch-sync/tombstones.json
{
  "deleted": [{
    "id": "e0c62e39...",
    "title": "Dashboard (old copy)",
    "reason": "Replaced by rename",
    "deletedAt": "2026-03-23"
  }]
}

How it works:

  • sync.sh skips tombstoned IDs
  • sync.sh status shows active screens
  • sync.sh status-all reveals hidden ones
  • Tombstones auto-prune when screens are deleted from Stitch

Why checked into git?
When someone tombstones a screen, the team sees it in the commit. "Why was this deleted?" — the reason field answers. Coordination via version control, not Slack messages.

Stitch uses UUID screen IDs — never reused. Safe to tombstone permanently.

Talking points (2 min)

"This is the most engineer-friendly concept in the whole pipeline. It's .gitignore for design screens."

"No delete API? Fine — we track deletions ourselves. The tombstone file is checked into git, so everyone on the team knows which screens are intentionally gone."

"Run status to see what's active. Run status-all to see the full picture including tombstoned. The sync script respects tombstones — it won't download dead screens."

"This is the kind of governance Figma gives you with team libraries and approvals. We built it with a JSON file and 20 lines of bash."
