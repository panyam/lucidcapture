Where Figma Still Wins

Stitch is getting better, but these are real gaps today:

  • Components with variants — Figma has size=sm/md/lg, state=default/hover/disabled. Stitch has none (yet)
  • Visual exploration speed — dragging beats prompting for rapid iteration
  • Design handoff — inspect mode, redlines, spacing overlays, asset export at 1x/2x/3x
  • Team governance — libraries, publishing, version history, branching, approvals
  • Prototyping — Figma prototypes with transitions and interactions for user testing

If you're a Figma power user, the MCP and tooling patterns from this talk still apply — imagine connecting Figma's API the same way. The principles are tool-agnostic.

Talking points (1.5 min)

"These are real strengths. Stitch is early — some of these will come. But the key insight: the MCP + skills + tombstoning patterns work with any design tool that has an API. Figma has a REST API too. The workflow is portable."
