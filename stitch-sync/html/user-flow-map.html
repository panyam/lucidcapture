# User Flow Map: Lucid Capture This document outlines the core user journey through the Lucid Capture application, connecting the screens currently on the canvas. ## 1. Landing Page to Dashboard - **Entry Point:** User arrives at the Landing Page (`{{DATA:SCREEN:SCREEN_4}}`). - **Action:** User clicks "Start Recording for Free" or "Login". - **Outcome:** Transitions to the **Dashboard** (`{{DATA:SCREEN:SCREEN_6}}`), where they can see their existing "Arcades". ## 2. Dashboard to Editor - **Action:** User clicks on an existing recording card (e.g., "Onboarding Flow") or the "Create New Arcade" button. - **Outcome:** Transitions to the **Editor** (`{{DATA:SCREEN:SCREEN_5}}`) to begin fine-tuning the demo. ## 3. Editor to Share Modal - **Action:** In the Editor, the user clicks the primary "Share" button in the top navigation bar. - **Outcome:** The **Share Modal** (`{{DATA:SCREEN:SCREEN_2}}`) opens as an overlay, providing options for links, embeds, and exports. ## 4. Share Modal Actions - **Action:** User clicks "Download File", "Copy Link", or "Done". - **Outcome:** "Done" returns the user to the Editor or Dashboard; other actions trigger their respective functional flows. --- **Next Steps:** I will now apply interactive hotspots to these screens so you can click through this flow directly on the canvas.
